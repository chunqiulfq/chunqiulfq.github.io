Linux 获取帮助

发表于 2014-04-13 | 更新于 2020-04-09 | 分类于 操作系统

  Linux 下的指令有太多太多, 我们不可能将所有的指令的用法都背下来记牢, 因此我们需要一些求助工具和说明文件, 这里以 RedHat 为例进行演示.

whatis

  查询的指令实现简短的说明.

<command> --help

  显示所查询指令的使用摘要以及参数列表, 蓝笔所画为使用摘要, 红笔所画为参数列表. 大部分指令可以使用 --help 参数, 但并不是所有, 有少部分指令不能使用.


man

  man 可以提供一些指令的说明文件, 在 redhat 里的每个指令都有相应的说明文件, 叫做 man page, man page 有像书一样的章节架构, 所有搜集来的 man page 叫做 Linux 操作手册.


  man 中一共有九个章节:

  1. User commands: 一般使用者使用的说明
  2. System calls: 系统调用
  3. Library calls: 库函数
  4. Gpecial files: 特殊文件
  5. File formats: 查询指令的档案格式说明
  6. Games: 游戏
  7. miscellaneous: 宏, 环境字符集等
  8. Adminstrative commonds: 只有 root 账户才能使用的指令说明
  9. kernel routines: 涉及kernel的一些东西

  其中 1, 5, 8 这三个比较重要, 离开 man page 输入 q. 使用 man -k <command> 可以查询包含 command 关键字的内容.



info

  提供更详细的指令说明, 使用章节一样的结构, 其中有一些节点, 可以连接其他章节. 进入 info 页面后按下 s 可以进入交互式查询, 离开按 q.


/usr/share/doc/

  可以查询一些额外的说明文件.


Red Hat documentation

  在网络上的说明文件: Red Hat documentation

Reading usage summaries

  在 --help 和 man 中都有使用摘要, 对比前后两张图就会发现它们中的使用摘要都是相同的.

Usage Summary 使用语法:

  1. 如果参数外面被 [ ] 括起来, 代表可有可无(optional)
  2. 如果参数外面被 <> 括起来, 代表参数是可变的
  3. 如果参数为 x|y|z, 代表只能使用 x, y, z 三者中的一个
  4. 如果参数为 -abc, 代表可以使用 a, y, z 三者中的任何值, 可以是一个, 两个, 三个
